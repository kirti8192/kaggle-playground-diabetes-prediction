# kaggle-playground-diabetes-prediction

Minimal repo structure for Kaggle Playground diabetes prediction.

## Structure
- `data/raw/`    Raw input files (ignored by git)
- `data/processed/`  Cleaned/feature data (ignored by git)
- `notebooks/`   Exploration notebooks
- `src/`         Reusable code (features, training, inference)
- `models/`      Trained model artifacts (ignored by git)
- `reports/`     Notes, figures, and analysis outputs
- `submissions/` Kaggle submission files

## Quickstart
1. Place dataset files in `data/raw/`.
2. Build features into `data/processed/`.
3. Train and save models in `models/`.
4. Generate predictions into `submissions/`.
